## 📊 FLOW PERUBAHAN

### BEFORE (BLOCKING)
```
Violation detected
         ↓
Screenshot capture (BLOCKING) - 2-5 seconds
         ↓
Upload to server (after screenshot done)
         ↓
If screenshot timeout → Upload fails silently
         ↓
Database: EMPTY ❌
```

### AFTER (NON-BLOCKING)
```
Violation detected
         ├→ Screenshot capture (async, background)
         │   └→ If fails, continue anyway
         │
         └→ Upload to server (IMMEDIATELY) ✅
             ├→ Send to /api/record-violation
             ├→ Server validates & inserts
             ├→ Database records violation ✅
             └→ Return violation_id
```

## 🎓 LEARNING POINTS

```
1. Promise chains can be problematic
   → Use async/await for clarity

2. Non-blocking operations essential
   → Screenshot in background, upload immediately

3. Detailed logging saves debugging time
   → [PREFIX] format makes filtering easy

4. Error handling must be explicit
   → Try-catch with detailed messages

5. State tracking prevents duplicates
   → Boolean flags work well

6. Timeout with AbortController
   → Prevents hanging requests
```
